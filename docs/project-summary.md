# 项目总结

## 1. 项目目标

本项目是一个面向公开视频内容的轻量视频下载网站，目标是先完成“粘贴链接 -> 解析信息 -> 选择成片格式 -> 下载保存”的核心闭环，并为后续扩展会员、字幕翻译、视频总结等能力预留空间。

当前阶段已经完成 Phase 1-3 的核心业务流程，能够在本地跑通前后端联调，并支持桌面端和移动端使用。

## 2. 当前技术栈

- 前端：React + Vite + TypeScript
- 样式：Tailwind CSS（通过 Vite 插件接入）
- 后端：FastAPI
- 通用视频解析与下载：yt-dlp
- 抖音专用解析：服务端自实现短链展开、分享页解析、无水印播放地址提取
- 媒体合并：ffmpeg

## 3. 已完成能力

### 3.1 核心下载闭环

- 支持输入公开视频链接并解析视频基础信息
- 支持展示标题、封面、作者、时长、平台信息
- 支持将底层格式流整理为普通用户可理解的“成片下载选项”
- 支持下载最终成片，而不是把 `audio only` / `video only` / 原始底层流直接暴露给用户

### 3.2 下载策略

- 统一采用“直链优先，代理兜底”的策略
- 对需要服务端处理的格式自动走代理下载
- 对音视频分离的平台输出合并后的 MP4 成片
- 对下载过程中的临时文件进行清理

### 3.3 平台支持现状

- 通用平台：继续通过 `yt-dlp` 支持 YouTube、Bilibili、Vimeo 等公开内容
- 抖音：已接入专用解析链路，不依赖用户手动提供 Cookie
- 抖音支持方式：
  - 短链 / 详情页链接解析
  - `iesdouyin` 公共接口优先
  - 分享页 `window._ROUTER_DATA` 兜底
  - `playwm -> play` 去水印播放地址转换

### 3.4 前端页面

- 已将首屏收敛为“导航 + 主标题 + 输入框 + 结果卡片”的结构
- 首屏重点聚焦下载主流程，其他说明内容已整体下沉到页面下方
- 下载区支持清晰度卡片选择和直接下载

## 4. 这次迭代中重点解决的问题

### 4.1 下载格式产品化

最初后端直接输出 `yt-dlp` 原始格式列表，普通用户看到的是一堆底层流、编码、协议和 `audio only / video only`，几乎不可用。

后续已经改为：

- 只展示用户可理解的成片选项
- 同分辨率下做归并与去重
- 默认推荐最高可用成片
- 合并型选项只通过代理下载输出 MP4

### 4.2 `m4s` 问题

之前某些平台下载结果会落成 `m4s` 或音视频分离文件。当前版本已修正为：

- 用户侧下载目标统一为成片
- 需要合并时由服务端完成
- 最终下载结果以 MP4 为主

### 4.3 缩略图防盗链问题

部分平台封面图在浏览器直接加载时会因为 Referer 或防盗链失败。当前版本已增加后端缩略图代理接口：

- `GET /api/video/thumbnail`

### 4.4 抖音下载失败问题

本轮已定位并修复一个关键 Bug：

- 某些抖音标题包含换行和特殊字符
- Windows 创建临时文件时会因为非法文件名失败
- 当前版本已对下载文件名进行清洗，避免因为标题内容导致落盘报错

## 5. 当前接口

### 5.1 `POST /api/video/parse`

作用：

- 解析视频基础信息
- 返回用户可下载的格式选项

### 5.2 `POST /api/video/download-link`

作用：

- 判断当前格式是否适合直链下载
- 返回下载策略信息

### 5.3 `GET /api/video/download`

作用：

- 服务端代理下载并返回文件

### 5.4 `GET /api/video/thumbnail`

作用：

- 代理拉取缩略图，规避浏览器端直接加载失败

## 6. 本地启动方式

### 6.1 启动后端

```powershell
cd backend
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload
```

### 6.2 启动前端

```powershell
cd frontend
npm run dev
```

默认地址：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:8000`

### 7.2 手工验证

已实际验证以下类型能力：

## 8. 当前边界

- 仅支持公开视频内容
- 不支持 DRM 内容
- 不支持需要登录态的私密内容
- 不接数据库
- 不包含账号系统
- 不包含真实支付
- 不包含批量下载任务系统
