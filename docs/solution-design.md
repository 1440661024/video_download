# 方案设计文档

## 技术选型

- 前端：React + Vite + Tailwind CSS
- 后端：FastAPI
- 下载核心：yt-dlp
- 运行方式：前后端分离，本地临时目录存储下载文件

## 架构说明

1. 用户在浏览器输入公开视频链接
2. React 前端请求 FastAPI `parse` 接口
3. FastAPI 调用 yt-dlp 提取视频信息和格式列表
4. 前端展示视频元信息与格式选项
5. 下载时先调用 `download-link` 判断是否可直链
6. 若可直链则直接跳转下载，否则回退到服务端 `download` 接口

## API 约定

### `POST /api/video/parse`

- 输入：`url`
- 输出：标题、封面、时长、作者、平台、格式列表、推荐下载策略

### `POST /api/video/download-link`

- 输入：`url`、`format_id`
- 输出：可用直链或服务端代理策略提示

### `GET /api/video/download`

- 输入：`url`、`format_id`
- 输出：服务端下载完成后将文件返回浏览器

## 下载策略

- 若目标格式存在稳定 HTTP 直链且文件不过大，优先使用直链
- 若为 m3u8、dash、音视频分离、疑似防盗链或大文件，则使用服务端代理
- 代理下载完成后自动清理临时文件
