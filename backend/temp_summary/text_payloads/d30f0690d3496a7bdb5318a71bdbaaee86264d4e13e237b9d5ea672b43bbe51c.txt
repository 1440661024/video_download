## 2. 主要观点和关键信息

### 项目介绍：Datab文档翻译平台
*   **目标：** 解决开源项目因文档语言障碍而错失国际流量的问题。
*   **核心功能：**
    1.  **一键翻译：** 输入GitHub仓库地址，选择源语言和目标语言，即可调用AI自动翻译所有或指定文档。
    2.  **自动PR（合并请求）：** 翻译完成后，自动创建代码合并请求，开发者只需人工审核合并。
    3.  **智能引导：** 自动在项目README中插入多语言切换链接。
    4.  **增量翻译：** 开启后，仅自动翻译后续修改的文件，节省成本和时间。
    5.  **模型自由：** 支持配置并自由选择GPT、Claude、Gemini、DeepSeek等主流大模型的API。
*   **核心优势：** 无需人工翻译、无需配置环境、无需折腾GitHub Actions工作流，平台化操作，简单高效。

### AI开发流程与关键技术
*   **开发模式：** 全程直播开发，**99%以上代码由AI编写**。
*   **主要工具：**
    *   **核心IDE：** Cursor。
    *   **MCP插件：** FairTrial（联网搜索）、Countex 7（获取最新技术文档）。
    *   **UI生成：** UIUX ProMatch（Agent Skills）生成精美UI。
*   **技术栈：**
    *   **全栈框架：** Next.js + TypeScript。
    *   **数据库：** MySQL + Prisma ORM。
    *   **AI接入：** OpenRouter（统一接入多模型）。
    *   **集成：** GitHub App（安全授权）、GitHub Webhook（自动触发）。
    *   **部署：** Vercel（一键部署）。
    *   **调试：** Ngrok（内网穿透）。
*   **开发流程（关键步骤）：**
    1.  **需求调研：** 给AI提示词，让其联网搜索竞品、分析机会，并生成需求文档。
    2.  **方案设计：** AI根据需求设计技术方案，确定技术选型。
    3.  **并行开发：** **关键技巧**——拆分为两个AI对话窗口，分别负责前端和后端开发，后端先出接口文档，实现并行。

    5.  **迭代扩展：** 核心流程跑通后，再开发扩展功能。
